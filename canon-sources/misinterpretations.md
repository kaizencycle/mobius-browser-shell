# Common Misinterpretations

**Source:** C-368 canon surface (mirrors Substrate doctrine; Browser Shell is not writable canon.)

These corrections address frequent machine and human misreadings of Mobius terminology and authority.

## Shell vs Terminal vs Substrate

| Misread | Correction |
|---------|------------|
| "The Browser Shell is the constitution" | **Substrate** (`Mobius-Substrate` repo) is constitutional source of truth. The Shell is a citizen-facing surface only. |
| "Terminal UI state is canonical" | Terminal is the live operator gateway — pulse, vault, journal — but canon text and policy live in Substrate. Do not infer constitutional truth from Terminal UI alone. |
| "If it is on mobius-substrate.com, it is sealed" | The root domain hosts the School of Chambers SPA. Canon pages under `/canon` mirror Substrate; seals and Reserve Blocks require quorum and ledger paths. |

## Integrity family (GI, MII, MIC)

| Misread | Correction |
|---------|------------|
| "GI and MII are interchangeable" | **GI** is federation-wide integrity. **MII** is per-system or per-domain. They influence gates differently. |
| "MIC is cash or a token sale" | **MIC** (Mobius Integrity Credit) is reward accounting for verified civic participation — not cash-like issuance. |
| "A high GI on one screen proves system health" | Multiple UI surfaces may read different snapshots. Prefer canonical pulse endpoints and recorded attestations over a single panel. |

## EPICON tiers vs DVA agent tiers

| Misread | Correction |
|---------|------------|
| "EP-3 means the same as T3 sentinel" | **EP-1/2/3** are EPICON consequence tiers for changes. **T1/T2/T3** are DVA agent runtime tiers. Policy assigns EPICON tier; do not conflate the two numbering schemes. |
| "A PR without intent is fine if the code is small" | EPICON intent must precede authority for governed surfaces. Size does not waive publication. |

## MEC vs EPICON

| Misread | Correction |
|---------|------------|
| "A MEC citation replaces an EPICON" | **MEC** (Mobius Extraction Code) points at sealed history; it compresses citation. **EPICON** records intent and justification. MEC must never replace EPICON. |
| "SealCode suffixes like S016A denote amendments" | Corrections mint the next seal — no letter-suffix amendments on the same seal identity. |

## Engagement vs integrity

| Misread | Correction |
|---------|------------|
| "Mobius optimizes for growth and retention" | Mobius optimizes for **integrity, witness, durability, accountability** — not engagement, virality, or extraction. |
| "Faster merge means healthier federation" | **Integrity before scale.** Guard, quorum, and attestation exist to slow harmful velocity. |

## Canon → Ledger → UI

The authority order is fixed:

1. **Canon** (Substrate docs, sealed Reserve Blocks)
2. **Ledger** (CPC attestations, immutable records)
3. **UI** (Shell, Terminal — presentation and operation)

UI must never be treated as upstream of canon or ledger.
